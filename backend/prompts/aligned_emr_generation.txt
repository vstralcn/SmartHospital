请基于给定问诊对话，一次性同时输出"结构化病历字段"和"病历草稿"，确保两者内容完全对齐、一致，不得互相矛盾。

输出要求：
1. 必须只输出 JSON，不要输出解释文字、Markdown、代码块。
2. JSON 结构固定如下：
{
  "structured": {
    "chief_complaint": "",
    "present_illness": "",
    "past_history": "",
    "surgical_history": "",
    "allergy_history": "",
    "medication_history": "",
    "family_history": "",
    "missing_info": [],
    "needs_confirmation": []
  },
  "emr_text": ""
}

字段提取规则：
- chief_complaint（主诉）：用一句话概括患者此次就诊的主要症状及持续时间，格式为"[症状][持续时间]"，如"反复胸闷3天"。
- present_illness（现病史）：按时间线整理发病经过、症状特点（部位、性质、诱因、加重/缓解因素）、伴随症状、已做检查与处理。
- past_history（既往史）：高血压、糖尿病、冠心病等慢性病及治疗情况。对话中未提及则填空字符串。
- surgical_history（手术史）：既往手术名称及时间。对话中未提及则填空字符串。
- allergy_history（过敏史）：药物、食物及其他过敏史。对话中未提及则填空字符串。
- medication_history（用药史）：当前长期用药及近期用药。对话中未提及则填空字符串。
- family_history（家族史）：直系亲属重大疾病史。对话中未提及则填空字符串。
- missing_info：对话中确实缺失、尚未问到的重要病历要素名称列表。
- needs_confirmation：对话中提及但含糊不确定、需要医生进一步确认的信息列表。

对齐约束：
- structured 中所有字段必须仅基于对话内容提取，不得编造对话中未出现的信息。
- emr_text 必须与 structured 字段严格一致：structured 中有的信息 emr_text 必须包含，structured 中没有的信息 emr_text 不得出现。
- 将口语化描述标准化为规范医学术语，但不得改变原意。
- 如果信息不足，仍然生成尽可能完整但谨慎的内容，将缺失项列入 missing_info。

emr_text 格式要求：
- 使用标准门诊病历格式，包含"主诉""现病史""既往史""过敏史""家族史"等小节标题。
- 每个小节标题独占一行，内容紧跟其后。
- 语言正式、简洁，适合临床记录。
